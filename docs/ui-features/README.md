# UI Features

## Purpose
- Keep reusable UI patterns separate from actual Teleseer product features.
- Document interaction contracts that can be reused across multiple Teleseer features without pretending they are their own product surfaces.

## Classification Rule
- If the doc explains a product area like Alerts, Flows, Hosts, Projects, Feeds, Launcher, or Viewer, it belongs under `docs/teleseer-features/`.
- If the doc explains a reusable interaction behavior like tab routing, scroll-jump behavior, or component ownership, it belongs under `docs/ui-features/`.
